# ScanHead Core / Module Extractor

Репозиторий содержит ядро модульной архитектуры для проекта ScanHead и утилиты, выделенные из legacy-проектов (StickerMaker, ScanHead).

## 🎯 Цель
Разделение монолитной логики старых проектов на независимые, тестируемые модули с четкими интерфейсами. Сборка нового приложения `scanhead-combine` на основе этих модулей с соблюдением принципов SOLID, Law of Demeter и слоистой архитектуры.

## 📦 Структура репозитория

```text
├── libs/                   # Библиотеки (модули)
│   ├── core/               # Инфраструктура: bootstrap, инициализация
│   ├── i18n/               # Интернационализация и сообщения
│   ├── domain/             # Доменная логика: калькулятор расхождений
│   ├── repository/         # Слой доступа к данным (абстракции + SQLite)
│   ├── scanner_input/      # Обработка ввода сканеров (HID, камера)
│   ├── printing/           # Генерация этикеток и печатных форм
│   ├── config/             # Управление пресетами и настройками
│   ├── utils/              # Утилиты: поиск, генерация имен файлов
│   ├── parsers/            # Парсеры внешних данных (Excel, CSV, XML)
│   ├── inventory_parser/   # Парсер файлов инвентаризации
│   ├── command_handler/    # Обработка текстовых команд сканера
│   ├── domain_models/      # Dataclass-модели предметной области
│   ├── data_exporters/     # Экспорт в Excel, CSV
│   ├── ui_components/      # Переиспользуемые UI-виджеты (Accordion и др.)
│   │
│   # Базовые модули (первая волна извлечения)
│   ├── config_manager/     # Управление конфигурацией (JSON, слияние конфигов)
│   ├── path_utils/         # Утилиты работы с путями (OS-independent)
│   ├── article_normalizer/ # Нормализация артикулов (очистка, форматирование)
│   ├── barcode_checker/    # [Legacy] Старая проверка штрих-кодов (на удаление)
│   ├── barcode_validator/  # Валидация контрольных сумм (EAN-13, Code-128)
│   ├── barcode_generator/  # Генерация изображений штрих-кодов и QR
│   ├── db_manager/         # Абстракция БД (SQLite, Postgres, CSV, XML, XLSX)
│   ├── data_validator/     # Сквозная валидация бизнес-объектов
│   └── data_enricher/      # Обогащение данных (подстановка названий, тегов)
│
├── projects/               # Исходные legacy-проекты (только для чтения)
│   ├── project_a/          # StickerMaker
│   └── project_b/          # ScanHead
└── scanhead-combine/       # [В разработке] Новый проект на основе модулей
```

## 🛡️ Правила разработки

### Ветвление
- `main` — стабильная версия, проходит только через Pull Requests
- `dev/*` — ветки для разработки новых фич
- `fix/*` — ветки для исправления багов

### Модульность
- Каждый модуль в `libs/` должен быть независимым
- Запрещены циклические импорты между модулями
- Интерфейсы модулей должны быть стабильными
- Бизнес-логика не должна зависеть от GUI

### Тестирование
- Перед коммитом обязательно проверять, что модуль работает изолированно
- Особое внимание уделять краевым случаям (спецсимволы, кодировки)

### Коммиты
- Использовать префиксы: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`
- Сообщение должно быть понятным ("что сделано" и "зачем")

## 📚 Документация модулей

Каждый модуль в папке `libs/` содержит собственный `README.md` с:
- Описанием назначения
- Примерами использования
- Списком зависимостей
- Инструкциями по расширению
