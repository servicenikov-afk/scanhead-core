# Scanner Input Module (libs/scanner_input)

Модуль обработки ввода от сканеров штрих-кодов. Абстрагирует работу с физическими устройствами (USB-HID) и будущими источниками (камера).

## Состав

### interfaces.py
Абстрактный базовый класс `ScannerProvider`. Определяет контракт для всех источников штрих-кодов:
- `start_scan()` — начать прослушивание
- `stop_scan()` — остановить
- `on_scan_result(callback)` — подписка на результаты

### hid_scanner.py
Реализация для USB-сканеров в режиме HID-клавиатуры:
- Буферизация потока символов
- Определение конца ввода по Enter
- Фильтрация артефактов и дребезга
- Поддержка русской кодировки

### camera_stub.py
Заглушка для будущего модуля камеры:
- Имитирует задержку сканирования
- Возвращает тестовые данные
- Реализует тот же интерфейс, что и HID-сканер

### buffer.py
Утилита буферизации ввода:
- Защита от частичного чтения данных
- Накопление символов до завершения команды

## Пример использования

```python
from libs.scanner_input import HidScanner

scanner = HidScanner()

def on_barcode(code):
    print(f"Отсканирован штрих-код: {code}")

scanner.on_scan_result(on_barcode)
scanner.start_scan()
```

## Зависимости
- Стандартная библиотека Python (threading, queue, typing)
- Для работы с HID: `evdev` (Linux) или `keyboard` (Windows/Mac) — опционально

## Расширение
Для добавления нового источника (например, камеры телефона):
1. Создайте класс, наследующий `ScannerProvider`
2. Реализуйте методы `start_scan()`, `stop_scan()`
3. Вызывайте callback при получении штрих-кода
